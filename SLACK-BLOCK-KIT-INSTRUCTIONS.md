# Instructions: Slack Block Kit Formatting for Sales AI Bot

> **For Claude Code.** This document tells you how to format ALL Slack bot responses using Block Kit with interactive elements. Every slash command response should look polished and professional — this is for an executive demo.

---

## CRITICAL RULES

1. **ALWAYS use Block Kit** for slash command responses (`/research`, `/deal`, `/coach`, `/draft-email`, `/pipeline`). Never send raw text for these.
2. **ALWAYS include a `text` fallback** alongside `blocks` — Slack notifications and some clients can't render blocks.
3. **Slack uses `mrkdwn`, NOT standard markdown.** Bold is `*text*`, NOT `**text**`. Links are `<https://url|display text>`, NOT `[text](url)`.
4. **Roleplay thread messages use plain `mrkdwn` only** — no blocks. Roleplay should feel conversational, not like a dashboard.
5. **The debrief at the end of a roleplay DOES use Block Kit** — it's a structured scorecard.

---

## BLOCK KIT BASICS

### How to send blocks in Slack Bolt

```python
# Always include both text (fallback) and blocks (rich)
say(
    text="Pipeline Health: $2.4M total, 3 deals at risk",  # Fallback for notifications
    blocks=[
        {"type": "header", "text": {"type": "plain_text", "text": "📈 Pipeline Health Digest"}},
        {"type": "section", "text": {"type": "mrkdwn", "text": "Your pipeline summary..."}},
    ],
    channel=channel_id,
)
```

### Available block types (use these)

| Block Type | What It Does | When to Use |
|-----------|-------------|-------------|
| `header` | Large bold text | Title of each response |
| `section` | Text paragraph, optionally with `fields` (2-column grid) or `accessory` (side element) | Most content |
| `section` with `fields` | 2-column key-value layout | Metrics, deal cards, summaries |
| `divider` | Horizontal line | Between sections |
| `context` | Small grey text | Timestamps, metadata, "Generated by..." |
| `actions` | Row of buttons, menus, date pickers | Interactive elements |
| `rich_text` | Advanced text with lists, quotes, formatting | Bullet lists (when needed) |

### mrkdwn syntax cheat sheet

```
*bold*                    — bold text
_italic_                  — italic text
~strikethrough~           — strikethrough
`inline code`             — inline code
```code block```          — code block
> quoted text             — blockquote
<https://url|Click here>  — hyperlink
<@U123ABC>                — mention user
<#C123ABC>                — mention channel
:emoji_name:              — emoji
\n                        — newline (use in strings)
```

def format_research_response(company_name, company_data, contacts, deals, news, rag_context):
    """Format a research brief as Block Kit."""

    # Build contacts string
    contacts_str = "\n".join([
        f"• *{c['name']}* — {c['title']} ({c['email']})"
        for c in contacts[:5]
    ])

    # Build deals string
    deals_str = "\n".join([
        f"• *{d['name']}* — {d['stage']} — ${d['amount']:,}"
        for d in deals[:5]
    ])

    # Build news string
    news_str = "\n".join([
        f"• {n['title']}" for n in news[:3]
    ])

    blocks = [
        {
            "type": "header",
            "text": {"type": "plain_text", "text": f"🔍 Research Brief: {company_name}"}
        },

        # Company overview in 2-column layout
        {
            "type": "section",
            "fields": [
                {"type": "mrkdwn", "text": f"*Industry:*\n{company_data.get('industry', 'N/A')}"},
                {"type": "mrkdwn", "text": f"*Size:*\n{company_data.get('size', 'N/A')} employees"},
                {"type": "mrkdwn", "text": f"*HQ:*\n{company_data.get('hq', 'N/A')}"},
                {"type": "mrkdwn", "text": f"*Revenue:*\n{company_data.get('revenue', 'N/A')}"},
            ]
        },

        {"type": "divider"},

        # Key contacts
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*👥 Key Contacts:*\n{contacts_str}"}
        },

        # Active deals
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*💼 Active Deals:*\n{deals_str}"}
        },

        {"type": "divider"},

        # Recent news
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*📰 Recent News:*\n{news_str}"}
        },

        # Talking points (from LLM)
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*💡 Talking Points:*\n{rag_context}"}
        },

        {"type": "divider"},

        # Action buttons
        {
            "type": "actions",
            "elements": [
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "📧 Draft Follow-Up Email"},
                    "action_id": "draft_email_from_research",
                    "value": company_name,
                },
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "🎯 Start Roleplay"},
                    "action_id": "start_roleplay_from_research",
                    "value": company_name,
                },
            ]
        },

        # Footer
        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": f"Generated by SalesCoach AI • Data from HubSpot + Tavily • {_timestamp()}"}
            ]
        },
    ]

    return blocks
```

def format_deal_response(deal, contacts, activities):
    """Format a deal status card as Block Kit."""

    # Stage emoji mapping
    stage_emoji = {
        "discovery": "🔎",
        "qualification": "📋",
        "demo scheduled": "📺",
        "demoscheduled": "📺",
        "proposal sent": "📄",
        "proposalsent": "📄",
        "negotiation": "🤝",
        "closedwon": "🎉",
        "closedlost": "❌",
    }
    emoji = stage_emoji.get(deal["stage"].lower().replace(" ", ""), "📊")

    # Build recent activity string
    activity_str = "\n".join([
        f"• _{a['date'][:10]}_ — {a['body'][:120]}..."
        if len(a['body']) > 120 else f"• _{a['date'][:10]}_ — {a['body']}"
        for a in activities[:4]
    ])

    blocks = [
        {
            "type": "header",
            "text": {"type": "plain_text", "text": f"{emoji} {deal['name']}"}
        },

        # Deal metrics in 2-column grid
        {
            "type": "section",
            "fields": [
                {"type": "mrkdwn", "text": f"*Stage:*\n{deal['stage']}"},
                {"type": "mrkdwn", "text": f"*Amount:*\n${int(deal['amount']):,}"},
                {"type": "mrkdwn", "text": f"*Close Date:*\n{deal['close_date']}"},
                {"type": "mrkdwn", "text": f"*Owner:*\n{deal.get('owner', 'Unassigned')}"},
            ]
        },

        {"type": "divider"},

        # Primary contact
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*👤 Primary Contact:*\n{contacts[0]['name']} — {contacts[0]['title']}\n{contacts[0]['email']}"}
        } if contacts else {
            "type": "section",
            "text": {"type": "mrkdwn", "text": "*👤 Primary Contact:*\n_No contact associated_"}
        },

        # Recent activity
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*📝 Recent Activity:*\n{activity_str}"}
        },

        {"type": "divider"},

        # Recommended next actions (from LLM)
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*✅ Recommended Next Actions:*\n{deal.get('recommendations', '_Analyzing..._')}"}
        },

        # Action buttons
        {
            "type": "actions",
            "elements": [
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "📧 Draft Email"},
                    "action_id": "draft_email_from_deal",
                    "value": deal["company"],
                },
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "🔍 Full Research"},
                    "action_id": "research_from_deal",
                    "value": deal["company"],
                },
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "📝 Log Note"},
                    "action_id": "log_note",
                    "value": deal["id"],
                },
            ]
        },

        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": f"Generated by SalesCoach AI • HubSpot data • {_timestamp()}"}
            ]
        },
    ]

    return blocks
```

### `/coach roleplay [scenario]` — Roleplay Start + Debrief

The roleplay itself uses plain mrkdwn in threads. Only the start message and the final debrief use Block Kit.

```python
def format_roleplay_start(persona, scenario):
    """Format the initial roleplay setup message (Block Kit)."""

    blocks = [
        {
            "type": "header",
            "text": {"type": "plain_text", "text": "🎯 Roleplay Session Starting"}
        },
        {
            "type": "section",
            "fields": [
                {"type": "mrkdwn", "text": f"*Scenario:*\n{scenario}"},
                {"type": "mrkdwn", "text": f"*Turns:*\n3-4 exchanges, then debrief"},
                {"type": "mrkdwn", "text": f"*You are:*\nThe sales rep"},
                {"type": "mrkdwn", "text": f"*I am:*\n{persona['name']}, {persona['title']}"},
            ]
        },
        {"type": "divider"},
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*Buyer Profile:*\n• *Name:* {persona['name']}\n• *Title:* {persona['title']} at {persona['company']}\n• *Personality:* {persona['personality']}\n\n_Reply in this thread to begin. I'll stay in character as {persona['name']}._"}
        },
        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": "💡 Tip: Treat this like a real call. Use discovery questions, handle objections, and try to advance the deal."}
            ]
        },
    ]

    return blocks


def format_roleplay_debrief(score, strengths, improvements, next_practice):
    """Format the final debrief scorecard (Block Kit)."""

    # Score color indicator
    if score >= 8:
        score_display = f"🟢 *{score}/10* — Excellent"
    elif score >= 6:
        score_display = f"🟡 *{score}/10* — Good, room to grow"
    else:
        score_display = f"🔴 *{score}/10* — Needs work"

    strengths_str = "\n".join([f"• {s}" for s in strengths])
    improvements_str = "\n".join([f"• {i}" for i in improvements])

    blocks = [
        {
            "type": "header",
            "text": {"type": "plain_text", "text": "🎯 Roleplay Debrief"}
        },

        # Score
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*Overall Score:* {score_display}"}
        },

        {"type": "divider"},

        # Strengths
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*✅ What You Did Well:*\n{strengths_str}"}
        },

        # Areas to improve
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*🔧 Areas to Improve:*\n{improvements_str}"}
        },

        {"type": "divider"},

        # Next practice suggestion
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*📝 Recommended Practice:*\n{next_practice}"}
        },

        # Action buttons
        {
            "type": "actions",
            "elements": [
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "🔄 Practice Again"},
                    "style": "primary",
                    "action_id": "practice_again",
                    "value": "same_scenario",
                },
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "🎯 Try Suggested Scenario"},
                    "action_id": "try_suggested",
                    "value": next_practice,
                },
            ]
        },

        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": f"SalesCoach AI • Scored against your sales playbook • {_timestamp()}"}
            ]
        },
    ]

    return blocks
```

### `/pipeline` — Pipeline Health Digest

```python
def format_pipeline_response(total_value, deal_count, weighted_forecast, deals_by_stage, flagged_deals, recommendations):
    """Format a pipeline health digest as Block Kit."""

    # Build stage breakdown
    stage_str = "\n".join([
        f"• *{stage}:* {count} deals — ${value:,}"
        for stage, count, value in deals_by_stage
    ])

    # Build flagged deals with risk type
    flag_emoji = {"stale": "⏰", "overdue": "🚨", "at_risk": "⚠️"}
    flagged_str = "\n".join([
        f"• {flag_emoji.get(d['flag'], '⚠️')} *{d['name']}* — {d['reason']}"
        for d in flagged_deals
    ]) or "_No flagged deals — looking good!_ 🎉"

    # Build recommendations
    rec_str = "\n".join([f"{i+1}. {r}" for i, r in enumerate(recommendations)])

    blocks = [
        {
            "type": "header",
            "text": {"type": "plain_text", "text": "📈 Pipeline Health Digest"}
        },

        # Top-line metrics in 2-column grid
        {
            "type": "section",
            "fields": [
                {"type": "mrkdwn", "text": f"*Total Pipeline:*\n${total_value:,}"},
                {"type": "mrkdwn", "text": f"*Open Deals:*\n{deal_count}"},
                {"type": "mrkdwn", "text": f"*Weighted Forecast:*\n${weighted_forecast:,}"},
                {"type": "mrkdwn", "text": f"*Flagged:*\n{len(flagged_deals)} deals"},
            ]
        },

        {"type": "divider"},

        # By stage
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*📊 By Stage:*\n{stage_str}"}
        },

        {"type": "divider"},

        # Flagged deals
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*⚠️ Flagged Deals:*\n{flagged_str}"}
        },

        {"type": "divider"},

        # Recommendations
        {
            "type": "section",
            "text": {"type": "mrkdwn", "text": f"*✅ Recommended Actions This Week:*\n{rec_str}"}
        },

        # Action buttons
        {
            "type": "actions",
            "elements": [
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "🔍 Drill into Flagged Deal"},
                    "action_id": "drill_flagged_deal",
                    "value": flagged_deals[0]["id"] if flagged_deals else "",
                },
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "🔄 Refresh Pipeline"},
                    "action_id": "refresh_pipeline",
                },
            ]
        },

        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": f"Generated by SalesCoach AI • HubSpot + BigQuery • {_timestamp()}"}
            ]
        },
    ]

    return blocks

## SLACK APP PERMISSIONS FOR INTERACTIVITY

For buttons and modals to work, you need these additional settings in the Slack app config:

1. **Enable Interactivity** — go to Features → Interactivity & Shortcuts → toggle ON. When using Socket Mode, the Request URL is handled automatically — leave it blank.

2. **Add Bot Token Scopes** (in addition to what's already configured):
   - `commands` — already set
   - `chat:write` — already set
   - Add: `users:read` — if you want to look up user names for the "Log Note" modal

3. **Socket Mode handles everything** — no need for a public Request URL. Button clicks, modal submissions, and slash commands all route through the same WebSocket connection.

## BLOCK KIT LIMITS TO KNOW

| Limit | Value |
|-------|-------|
| Max blocks per message | 50 |
| Max text in a section block | 3,000 characters |
| Max fields in a section | 10 (but use 4-6 for clean 2-column layout) |
| Max buttons in an actions block | 25 (but use 2-4 for clarity) |
| Max characters in button text | 75 |
| Max characters in header text | 150 |
| Max characters in context text | 3,000 |
| `action_id` must be unique | Per message (not globally) |
| `value` field on buttons | Max 2,000 characters (use for passing data like company names or JSON) |

## FILE ORGANIZATION

Put all the formatting functions in a dedicated file:

```
sales-ai-agents/
├── formatters/
│   ├── __init__.py
│   ├── research.py          # format_research_response()
│   ├── deal.py              # format_deal_response()

from formatters.research import format_research_response

## QUICK REFERENCE: BLOCK JSON PATTERNS

### Header
```json
{"type": "header", "text": {"type": "plain_text", "text": "Title Here"}}
```

### Section with text
```json
{"type": "section", "text": {"type": "mrkdwn", "text": "*Bold* and _italic_ text"}}
```

### Section with 2-column fields (max 10 fields)
```json
{"type": "section", "fields": [
    {"type": "mrkdwn", "text": "*Label:*\nValue"},
    {"type": "mrkdwn", "text": "*Label:*\nValue"}
]}
```

### Divider
```json
{"type": "divider"}
```

### Buttons (primary = green, danger = red, default = grey)
```json
{"type": "actions", "elements": [
    {"type": "button", "text": {"type": "plain_text", "text": "Confirm"}, "style": "primary", "action_id": "confirm", "value": "data"},
    {"type": "button", "text": {"type": "plain_text", "text": "Cancel"}, "style": "danger", "action_id": "cancel"}
]}
```

### Context (small grey footer text)
```json
{"type": "context", "elements": [
    {"type": "mrkdwn", "text": "Small footer text here"}
]}
```

### Overflow menu (three-dot menu on a section)
```json
{"type": "section", "text": {"type": "mrkdwn", "text": "Content"}, "accessory": {
    "type": "overflow", "action_id": "overflow_menu",
    "options": [
        {"text": {"type": "plain_text", "text": "Option 1"}, "value": "opt1"},
        {"text": {"type": "plain_text", "text": "Option 2"}, "value": "opt2"}
    ]
}}
```
