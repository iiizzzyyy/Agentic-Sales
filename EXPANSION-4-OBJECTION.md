## CONTEXT — WHAT AND WHY

A rep is on a live call. The buyer says "your pricing is too high." The rep needs a response **right now** — not in 15 seconds. `/objection` is the fastest command in the bot: RAG lookup + LLM response, optimized for speed.

This is the "AI copilot in real-time" moment. It shows the AI helps reps **during** conversations, not just before/after.

**This is the easiest expansion to build** — it's essentially a RAG search + LLM format. Should take ~30 minutes.

---

## WHAT IT DOES

1. Rep types `/objection your pricing is too high` (during a live call)
2. Bot immediately searches RAG for:
   - Objection handling playbook chunks matching this objection
   - Battle cards with pricing positioning
   - Relevant case studies (proof points)
3. LLM generates a concise, ready-to-use response
4. Bot posts in 3-5 seconds with: the response to use, supporting data points, and what NOT to say

## STEP 1: Create `formatters/objection.py`

```python
"""Block Kit formatting for /objection command."""
from formatters.helpers import timestamp


def format_objection_response(
    objection: str,
    response_script: str,
    proof_points: list,
    avoid: list,
    follow_up_question: str,
) -> tuple[list, str]:
    """Format an objection handling response as Block Kit.

    Args:
        objection: The original objection text
        response_script: LLM-generated response the rep can use verbatim
        proof_points: List of supporting data points / case study references
        avoid: List of things NOT to say
        follow_up_question: A question to pivot the conversation

    Returns:
        tuple: (blocks, fallback_text)
    """
    # Proof points
    if proof_points:
        proof_str = "\n".join([f"• {p}" for p in proof_points[:4]])
    else:
        proof_str = "_No specific proof points available_"

    # What to avoid
    if avoid:
        avoid_str = "\n".join([f"• ❌ {a}" for a in avoid[:3]])
    else:
        avoid_str = "_No specific warnings_"

    blocks = [
        {
            "type": "header",
            "text": {"type": "plain_text", "text": "🛡️ Objection Response", "emoji": True}
        },
        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": f"They said: _{objection}_"}
            ]
        },

        {"type": "divider"},

        # The response to use — this is the star of the show
        {
            "type": "section",
            "text": {
                "type": "mrkdwn",
                "text": f"*💬 Say This:*\n\n{response_script}"
            }
        },

        {"type": "divider"},

        # Two columns: proof points and what to avoid
        {
            "type": "section",
            "fields": [
                {"type": "mrkdwn", "text": f"*📊 Proof Points:*\n{proof_str}"},
                {"type": "mrkdwn", "text": f"*🚫 Avoid Saying:*\n{avoid_str}"},
            ]
        },

        {"type": "divider"},

        # Follow-up question to pivot
        {
            "type": "section",
            "text": {
                "type": "mrkdwn",
                "text": f"*🔄 Pivot With:*\n_{follow_up_question}_"
            }
        },

        # Action buttons
        {
            "type": "actions",
            "elements": [
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "Practice This Objection", "emoji": True},
                    "action_id": "practice_objection",
                    "value": objection,
                },
                {
                    "type": "button",
                    "text": {"type": "plain_text", "text": "More Detail", "emoji": True},
                    "action_id": "expand_objection",
                    "value": objection,
                },
            ]
        },

        {
            "type": "context",
            "elements": [
                {"type": "mrkdwn", "text": f"Generated by SalesCoach AI • Objection Playbook • {timestamp()}"}
            ]
        },
    ]

    fallback = f"Objection Response: {objection}"
    return blocks, fallback
```

## STEP 3: Add `/objection` Command Handler to `app.py`

**Key design choice:** This command should be FAST. Use only RAG (no web search, no MCP) to minimize latency. The LLM prompt should be concise.

```python
@app.command("/objection")
def handle_objection(ack, say, command):
    """Instant objection handling response — optimized for speed."""
    ack()
    objection_text = command["text"].strip()
    channel_id = command["channel_id"]
    user_id = command["user_id"]

    if not objection_text:
        say(
            "Tell me the objection: `/objection [what they said]`\n"
            "Examples:\n• `/objection your pricing is too high`\n"
            "• `/objection we're already using a competitor`\n"
            "• `/objection we need to think about it`",
            channel=channel_id,
        )
        return

    say(f"🛡️ Finding response...", channel=channel_id)

    # 1. RAG search — objection playbook + battle cards (fast, no API calls)
    from tools.rag import search_playbooks
    objection_context = search_playbooks(
        f"objection handling: {objection_text}",
        n_results=5,
    )
    battle_card_context = search_playbooks(
        f"competitive positioning value proposition pricing",
        n_results=3,
    )

    # 2. LLM — concise prompt for fast response
    result = app_graph.invoke({
        "messages": [HumanMessage(content=f"""A sales rep is on a LIVE CALL. The buyer just said: "{objection_text}"

The rep needs a response RIGHT NOW. Be concise and actionable.

OBJECTION HANDLING PLAYBOOK:
{objection_context}

BATTLE CARDS / VALUE PROP:
{battle_card_context}

Generate EXACTLY this format:

RESPONSE: [2-3 sentence response the rep can say verbatim. Acknowledge the concern, reframe, provide value.]

PROOF_POINT_1: [One specific data point, case study reference, or ROI number]
PROOF_POINT_2: [Another proof point]
PROOF_POINT_3: [Another proof point]

AVOID_1: [One thing NOT to say — a common mistake]
AVOID_2: [Another thing to avoid]

PIVOT_QUESTION: [One question to redirect the conversation productively]

Be specific, not generic. Use the playbook context.""")],
        "route": "assistant",
        "workflow": "research",
        "user_id": user_id,
        "channel_id": channel_id,
        "thread_ts": None,
    })

    llm_response = result["messages"][-1].content

    # 3. Parse the structured response
    response_script = ""
    proof_points = []
    avoid = []
    follow_up = ""

    for line in llm_response.split("\n"):
        line = line.strip()
        if line.upper().startswith("RESPONSE:"):
            response_script = line.split(":", 1)[1].strip()
        elif line.upper().startswith("PROOF_POINT"):
            proof_points.append(line.split(":", 1)[1].strip() if ":" in line else line)
        elif line.upper().startswith("AVOID"):
            avoid.append(line.split(":", 1)[1].strip() if ":" in line else line)
        elif line.upper().startswith("PIVOT_QUESTION:"):
            follow_up = line.split(":", 1)[1].strip()

    # Fallback if parsing fails — use the whole LLM response as the script
    if not response_script:
        response_script = llm_response[:500]

    if not follow_up:
        follow_up = "What would make this a no-brainer for you?"

    # 4. Format and send
    from formatters import format_objection_response
    blocks, fallback = format_objection_response(
        objection=objection_text,
        response_script=response_script,
        proof_points=proof_points,
        avoid=avoid,
        follow_up_question=follow_up,
    )

    say(text=fallback, blocks=blocks, channel=channel_id)
```

@app.action("practice_objection")
def handle_practice_objection(ack, body, say):
    """Start a roleplay focused on this specific objection."""
    ack()
    objection = body["actions"][0]["value"]
    channel = body["channel"]["id"]

    say(
        text=f"🎯 Try: `/coach roleplay objection handling — buyer says: {objection}`",
        channel=channel,
    )


@app.action("expand_objection")
def handle_expand_objection(ack, body, say):
    """Get a more detailed analysis of this objection."""
    ack()
    objection = body["actions"][0]["value"]
    channel = body["channel"]["id"]
    user_id = body["user"]["id"]

    say(f"📖 Getting deeper analysis of this objection...", channel=channel)

    # Run a more detailed LLM analysis
    from tools.rag import search_playbooks
    context = search_playbooks(f"objection handling {objection}", n_results=8)

    result = app_graph.invoke({
        "messages": [HumanMessage(content=f"""Provide an in-depth analysis of this sales objection: "{objection}"

PLAYBOOK CONTEXT:
{context}

Include:
1. Why buyers typically raise this objection (root causes)
2. Three different response approaches (empathetic, data-driven, redirect)
3. Real-world examples of how top reps handle this
4. How to prevent this objection earlier in the sales process

Keep it practical and actionable.""")],
        "route": "assistant",
        "workflow": "research",
        "user_id": user_id,
        "channel_id": channel,
        "thread_ts": None,
    })

    say(text=result["messages"][-1].content, channel=channel)

## PERFORMANCE NOTE

This command should respond in **3-5 seconds** because:
- RAG search is local (ChromaDB, sub-second)
- No web search or MCP calls
- LLM prompt is short and structured

If latency is an issue, consider using a faster model for this specific command (e.g., `anthropic/claude-haiku-3.5` via OpenRouter) since it doesn't need deep reasoning.
